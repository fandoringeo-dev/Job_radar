# Job Radar — заметки (NOTES)

## Как устроены папки
- `src/` — код приложения (мы будем импортировать пакеты отсюда).
- `app/` — “главный пакет” проекта (модули внутри него импортируются как `from app...`).
- `core/` — общие утилиты (логирование, конфиг, клиент HTTP).
- `scrapers/` — сборщики по источникам вакансий (1 сайт = 1 модуль).
- `models/` — модели данных (Vacancy и т.п.).
- `storage/` — локальные данные (БД, временные файлы). Не коммитим.
- `config/` — настройки источников и проекта.
- `DEVLOG.md` — что сделали и зачем.


## Как понять, что venv активен
- В терминале часто появляется префикс `(.venv)`, но это не 100% гарантия.
- Надёжная проверка: `python -c "import sys; print(sys.executable)"` — путь должен быть внутри `.venv\Scripts\python.exe`.
- Команды `pip install ...` ставят пакеты в активное окружение, поэтому важно активировать venv перед установкой зависимостей.


## Архитектура по графу импортов

Схема (docs/import_graph.png) показывает зависимости:

- app.main
  - импортирует app.core.logging (настройка логов).
  - импортирует app.scrapers.habr_search (источник вакансий).

- app.scrapers.habr_search
  - импортирует app.models.job (создаёт Job).

- app.scrapers.rss_search
  - тоже импортирует app.models.job (потенциальный источник №2).

Инструменты:
- app.tools.import_map — текстовая карта импортов.
- app.tools.import_graph — генерирует картинку import_graph.png.
